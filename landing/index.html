✦ FAQ

Questions, answered.

Is it truly invisible to my interviewer? +
Yes. Ghostnote uses the Windows SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE) API which operates at the compositor level. Any screen recording, screensharing, or screenshot tool — including Zoom, Teams, Google Meet, OBS, Loom, and Windows built-in Snipping Tool — will capture a completely blank/transparent area where Ghostnote is displayed. You can verify this yourself by sharing your screen and looking at the recording.
How does it capture the interview audio without a bot? +
Ghostnote uses Windows WASAPI loopback capture — it reads directly from your system's audio output buffer, not from a microphone. This means it captures everything playing through your speakers or headphones: the interviewer's voice from the Zoom call, your own voice via audio echo, and anything else playing on your system. No meeting bot is ever joined. No one else is ever invited to your call.
Which AI models does it use? +
Ghostnote supports both gpt-4o-mini (default — fastest, most cost-efficient) and gpt-4o (Pro — highest quality reasoning). Both use OpenAI's streaming API so answers appear word-by-word instead of waiting for the full response. Transcription is powered by Deepgram's nova-2 model, which provides near-human accuracy.
Does it work on Mac or Linux? +
Currently Ghostnote is Windows-only. The screen-capture invisibility feature relies on WDA_EXCLUDEFROMCAPTURE, a Windows-exclusive Win32 API. WASAPI loopback audio capture is also Windows-specific. Mac support using kCGWindowSharingNone and CoreAudio is planned for a future release.
Is my data stored or shared with anyone? +
Audio is captured locally and streamed directly to Deepgram's servers for transcription over an encrypted WebSocket — it is never stored. AI conversations are sent to OpenAI's API and are subject to their retention policy. Session metadata (word count, duration) is optionally synced to your account in AWS DynamoDB. Nothing is ever sold to third parties.
Do I need my own API keys? +
For the self-hosted free version, yes — you'll need an OpenAI API key and a Deepgram API key (both have free tiers). Add them to a .env.local file and you're ready. The Pro plan includes hosted API keys so you don't need to manage your own.
