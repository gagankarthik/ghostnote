How it works

From audio to insight in under a second

Four systems working in real time, invisible to everyone in the meeting.

1
WASAPI · Rust
Audio captured locally
Rust captures system audio via WASAPI loopback — the same audio going to your speakers. No microphone access required. Chunked every 250ms.
2
Deepgram Nova-2
Live transcript streams in
Audio chunks flow to Deepgram's streaming WebSocket. Interim words appear instantly; final segments committed. ~300ms end-to-end latency.
3
GPT-4o · Every 15s
AI reads the last 60 seconds
The last 60 seconds of transcript is sent to GPT-4o with a focused prompt. 2–3 actionable bullets appear in the overlay within 2 seconds.
4
Tauri · Windows API
Suggestions appear silently
The overlay renders using WDA_EXCLUDEFROMCAPTURE. Zoom sees nothing. You see everything.

Privacy & Security

Built for confidential conversations

No audio stored. No recording uploaded. Everything you say stays between you and the AI.

Invisible at the OS level
WDA_EXCLUDEFROMCAPTURE is a Windows kernel-level API. No screen-recording software can override it — not Zoom, not OBS, not any third-party tool.
Encrypted transit, zero storage
Audio streams to Deepgram over WSS (encrypted WebSocket). Transcripts are processed in memory and never written to disk. Deepgram's data retention: zero.
No microphone access
WASAPI loopback captures what's already playing through your speakers — no microphone permission needed. Your physical mic is never touched.
